# ADR 001: Custom HMAC token вместо JWT

**Статус:** Accepted  
**Дата:** 2026-01-15  
**Автор:** Ситников Сергей

## Контекст

Нужна аутентификация для тестового SUT. Варианты:
1. JWT (python-jose / PyJWT)
2. Custom HMAC-SHA256 токен

## Решение

Используем custom HMAC-SHA256 токен формата `username:expires_iso:signature`.

## Обоснование

**За custom token:**
- Нулевые внешние зависимости для SUT
- Полный контроль над форматом — удобно тестировать edge cases
- Достаточно для демонстрационного проекта

**Против JWT:**
- Дополнительная зависимость (PyJWT/python-jose)
- Overhead для проекта где фокус — тесты, не продакшн-auth

## Последствия

- `_create_token` / `_verify_token` — самописные функции в `auth.py`
- Тесты могут создавать токены с произвольными параметрами (удобно для негативных тестов)
- **В production: заменить на PyJWT + RS256**

## Альтернативы рассмотрены

- `python-jose`: избыточно для SUT
- `authlib`: слишком большая библиотека
